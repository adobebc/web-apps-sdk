Create / Update a file

Uploads a file in a specified folder.

Request

  • Method: PUT
  • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the Authorize your API calls document for more info on how this URL is formed.
    • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
  • Path: /api/v2/admin/sites/[siteId]/storage/[filePath]?version=draft
  • Authorization header: This should contain the authorization token. Here is how to obtain the token.
  • Required Permissions: Can use SFTP & File Manager

Parameters

File contents as binary

Response

Returns the status of the new file.

Notes

Mandatory header {% raw %}Content-Type: application/octet-stream{% endraw %}

Datacenter location

If the url for the PUT request does not correspond to the datacenter where the site is located, a 301 response will be returned that will redirect the request to the correct location.

Nested folders

If you use a filePath with nested folders, the API will automatically create the nested folder structure.

The depth of the nested folder structure is limited only by the URL length.

Query params

The API supports an optional query param named version that as the following possible values:

  • draft
    • If the file is a page, than a drat of it will be created/updated. The live version will not be touched.
    • If the file is not a page, than this will be ignored
  • draft-publish
    • If the file in question is a page than the content will first be saved as a draft, overwriting the current one and a publish operation will be done at the end.
    • If the file in question is not a page, that this will be ignored.

Accepts and returns JSON as Content-Type.

Request:

{% raw %}PUT /api/v2/admin/sites/current/storage/images/test.txt HTTPS/1.1
Accept: application/json
ContentType: "application/octet-stream"
Authorization: c50f6e6be0d1481ca0d8eb0c63642fdd171c17846af04cdd95676a0888141f73
{% endraw %}

Response:

{% raw %}HTTP/1.1 200 OK
{% endraw %}

Sample code

Create a new file in the root folder

{% raw %}var request = $.ajax({
    url: "/api/v2/admin/sites/current/storage/testing.txt?version=draft-publish",
    type: "PUT",
    headers: {
        "Authorization": $.cookie('access_token')
    },
    contentType: "application/octet-stream",
    data: "The file's content goes here"
});
{% endraw %}

Create file (specify the full path)

    headers: {
        "Authorization": $.cookie('access_token')
    },

    data: 'New content'
});        
      
{% endraw %}

Upload files from your computer

{% raw %}<input type="file" id="datafile">
<input type="button" onclick = "attachment()" value="Send">
<script>
var access_token = BCAPI.Helper.Site.getAccessToken();
function attachment() {
            
	var fileContent = $("#datafile").get(0).files[0];
            
	var request = $.ajax({
	    url: "/api/v2/admin/sites/current/storage/my/large_video.avi", //change this to reflect the name of your file
	    type: "PUT",
	    headers: {
	        "Authorization": $.cookie('access_token')
	    },
    	contentType: "application/octet-stream",        
    	data: fileContent,
        processData: false
	}); 

    var progressChecker = window.setInterval(function() {
    	_checkUploadStatus();
    }, 500);    
    
    request.done(function() {
        window.clearInterval(progressChecker);
    });
};
</script>{% endraw %}

Notes:

  • A file is created in your site's file system only after adding some content.
  • The content can be any javascript object, including file objects obtained from html upload forms.
  • If you omit the {% raw %}/{% endraw %} at the beginning of the file path, the system will add this.

Error codes

This method will return the following error codes:

  • {% raw %}200{% endraw %} - OK
  • {% raw %}301{% endraw %} - Redirect to correct data center url
  • {% raw %}400{% endraw %} - Bad request:
    • {% raw %}104000{% endraw %} - Generic FileAPI error
    • {% raw %}104002{% endraw %} - Not a file
    • {% raw %}104003{% endraw %} - Not enough privileges
    • {% raw %}104004{% endraw %} - Invalid login multiple sites
    • {% raw %}104005{% endraw %} - Exceeded maximum file size
    • {% raw %}104006{% endraw %} - Invalid editable region name, if a template
    • {% raw %}104007{% endraw %} - Can't find tag {pagecontent} in editable region, if a template
    • {% raw %}104008{% endraw %} - Can't find tag {pagecontent}, if a template
    • {% raw %}104009{% endraw %} - Path not found
    • {% raw %}104010{% endraw %} - Destination path not found
    • {% raw %}104012{% endraw %} - Web page in workflow updated, if trying to update a page in a workflow
    • {% raw %}104014{% endraw %} - Name required
    • {% raw %}104015{% endraw %} - Invalid URL
    • {% raw %}104016{% endraw %} - Name not unique
    • {% raw %}104019{% endraw %} - Can't find folder
    • {% raw %}104022{% endraw %} - Destination exists
    • {% raw %}104023{% endraw %} - Unauthorized access
    • {% raw %}104028{% endraw %} - File or folder name too long
  • {% raw %}401{% endraw %}- when authentication token is incorrect
