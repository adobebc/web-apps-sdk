new ItemCategory()

This class provides a way of retrieving and assigning categories to items as an array of category id-s that can be retrieved via the BCAPI.Models.CategoryCollection

Source:

  • src/webapp_item_category.js

Examples

{% raw %}var itemCategories = new BCAPI.Models.WebApp.ItemCategory(WEBAPP_NAME, ITEM_ID);
itemCategories.fetch({
    success: function (data) {
        _.each(data.get("items"), function (categoryId) {
            var category = new BCAPI.Models.Category({
                id: categoryId
            });
            category.fetch({
                success: doSomethingWithCategName,
                error: onError
            })
        })
    },
    error: function (data, xhr) {}
});{% endraw %}

To assign multiple categories:

{% raw %}var itemCategories = new BCAPI.Models.WebApp.ItemCategory(WEBAPP_NAME, ITEM_ID);
itemCategories.set('items', [1, 2, 3, 4]);
itemCategories.save({
    success: onSaveOK,
    error: onSaveFailed
})
{% endraw %}

Error Codes

This method will return the following error codes:

  • {% raw %}200{% endraw %} - success
  • {% raw %}400{% endraw %} - bad request
    • {% raw %}200000{% endraw %} - An unspecified error has occured
    • {% raw %}200009{% endraw %} - Category id is invalid
  • {% raw %}401{% endraw %} - unauthorized - when the Authorization header is not present, or contains an invalid site token
  • {% raw %}403{% endraw %} - forbidden - when the user does not have Edit WebApp Item permission
  • {% raw %}404{% endraw %} - not found:
    • {% raw %}190001{% endraw %} - The web app could not be found (the webAppName param from the URL does not match)
    • {% raw %}200001{% endraw %} - The webapp item was not found (the webAppName and webAppItemId from the URL do not match)
