Create Web App

Create a new Web App

Request

  • Method: POST
  • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the Authorize your API calls document for more info on how this URL is formed.
    • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
  • Path: /api/v2/admin/sites/current/webapps
    • Alternatively use siteID instead of 'current'
  • Authorization header: This should contain the authorization token. Here is how to obtain the token.
  • Required Permissions: Create Web Apps

Parameters

  • {% raw %}name{% endraw %} - required (string)
  • {% raw %}slug{% endraw %} - (string, optional) - it defaults to the name of the web app, or a variation on the name, if a slug with the name already exists
  • {% raw %}allowFileUpload{% endraw %} - . Defaults to false (boolean, optional)
  • {% raw %}uploadFolder{% endraw %} - required only if allowFileUpload is True. Must be a valid absolute path in the site, created in advance (string)
  • {% raw %}disableDetailPages{% endraw %}- (boolean, optional)
  • {% raw %}locationEnabled{% endraw %} - (boolean, optional)
  • {% raw %}anyoneCanEdit{% endraw %} - (boolean, optional)
  • {% raw %}customerCanAdd{% endraw %} - (boolean, optional)
  • {% raw %}customerCanEdit{% endraw %} - (boolean, optional)
  • {% raw %}customerCanDelete{% endraw %} - (boolean, optional)
  • {% raw %}requiresPayment{% endraw %} - (boolean, optional)
  • {% raw %}requiresApproval{% endraw %} - (boolean, optional)
  • {% raw %}roleId{% endraw %} - optional, defaults to -1 (integer)
  • {% raw %}templateId{% endraw %} - optional, defaults to -1 (integer)
  • {% raw %}validDays{% endraw %} - optional, defaults to -1 (never expire) (integer)

Response

Returns an empty body and a location header containing the URI to retrieve category details.

Examples

Accepts and returns JSON as Content-Type.

Request:

{% raw %}POST /api/v2/admin/sites/current/webapps HTTPS/1.1
Content-Length: 57
Connection: keep-alive
Content-Type: application/json

{"name": "FirstWebAppFromApi"}

{% endraw %}

Response:

{% raw %}HTTP/1.1 201 OK
Location: /v2/admin/sites/current/webapps/FirstWebAppFromApi
Content-Type: application/json; charset=utf-8
Content-Length: 0
{% endraw %}

Notes

  • Creating a Web App does not allow passing in the custom fields. To add custom fields, use the Custom Fields API endpoint. Creating a Web App will create (or preserve) the default layouts (list, alternate, detail, edit)
  • if passing a value for the uploadFolder parameter that does not exist on the file system, the system will attempt to create the folder, and its entire path. It reuses the mechanism used in the FileSystem APIs to create a folder, so the same exceptions get thrown.
  • Only the webapp Name parameter is mandatory. The rest have defaults, as follows:
    • templateId - -1
    • uploadFolder - -1
    • slug - the name of the web app or a variation of the name
    • requiresApproval - true
    • allowFileUpload - false
    • customerCanAdd, customerCanDelete, customerCanEdit, anyoneCanEdit - false
    • requiresPayment - false
    • validDays - -1 (never expire)
    • roleId - 0
    • hasAddress - false
    • disableDetailPages - false
    • locationEnabled - false
  • The webapp name must not be empty and can only contain valid characters, that can be later used on the file system and the API uris. The invalid characters are: / ? # % & * + \ | : ; . â€œ > < If you pass in values for itemSystemFields and / or itemFields they will be ignored.

Error codes

This method will return the following error codes:

  • {% raw %}201/Created{% endraw %} - Web App successfully created
  • {% raw %}400/Bad request{% endraw %} - Invalid request. Sub-codes are:
    • {% raw %}190000{% endraw %} - Generic error
    • {% raw %}190002{% endraw %} - Invalid slug
    • {% raw %}190003{% endraw %} - Slug URL too long
    • {% raw %}190004{% endraw %} - slug already exists
    • {% raw %}190005{% endraw %} - webapp with same name already exists
    • {% raw %}190009{% endraw %} - invalid webapp name; see notes section below for list of invalid characters
    • {% raw %}190010{% endraw %} - invalid template id
  • {% raw %}401{% endraw %} - unauthorized - when the Authorization header is not present, or contains an invalid site token
    • {% raw %}101000{% endraw %} - sub-error code
  • {% raw %}403{% endraw %} - Forbidden : this is returned when the user trying to access the API does not have the proper permissions
  • {% raw %}404{% endraw %} - Not Found
    • {% raw %}190001{% endraw %} - Not Found
