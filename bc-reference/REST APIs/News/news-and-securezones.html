News and securezones

    List the secure zones a news item is added into

    Request

    • Method: GET
    • Version: v3, this API call supports additional sort and filter mechanisms as well as retrieving only a sub-set of fields

      Note: In order to access the v3 API endpoints please make sure you have enabled the new rendering engine.

    • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the Authorize your API calls document for more info on how this URL is formed.
      • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
    • Path: /webresources/api/v3/sites/current/news/[news_ID]/securezones
      • Alternatively, use siteID instead of 'current'
    • Authorization header: This should contain the authorization token. Here is how to obtain the token.

    Response

    An array of the first 10 securezones assigned to the news item; each object has the following properties:

    • {% raw %}id{% endraw %} - the id of the secure zone (integer)
    • {% raw %}siteId{% endraw %} - the id of the site (integer)
    • {% raw %}landingPageId{% endraw %} - the ID of the secure zone's landing page (string)
    • {% raw %}name{% endraw %} - the name of the secure zone (string)
    • {% raw %}sellAccess{% endraw %} - whether you charge the customers who get access to the secure zone or not (boolean)
    • {% raw %}costPerPeriod{% endraw %} - the membership cost amount (integer)
    • {% raw %}orderCountryCode{% endraw %} - the currency to use when charging the customers for access(string)
    • {% raw %}cycleType{% endraw %} - the frequency of membership renewal (integer)
      • "1" Once
      • "2" Daily
      • "3" Weekly
      • "4" Fortnightly
      • "5" Monthly
      • "6" Quarterly
      • "7" Half Yearly
      • "8" Yearly
    • {% raw %}createDate{% endraw %} - the date the secure zone was created (string)
    • {% raw %}lastUpdateDate{% endraw %} - the date the secure zone was last updated (string)

    You can output additional fields to those listed above as default, see the API v3 Discovery APP for additional details.

    Accepts and returns JSON as Content-Type.

    Add a news item to a secure zone

    Request

    • Method: POST
    • Version: v3, this API call supports additional sort and filter mechanisms as well as retrieving only a sub-set of fields

      Note: In order to access the v3 API endpoints please make sure you have enabled the new rendering engine.

    • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the Authorize your API calls document for more info on how this URL is formed.
      • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
    • Path: /webresources/api/v3/sites/current/news/[news_ID]/securezones
      • Alternatively, use siteID instead of 'current'
    • Authorization header: This should contain the authorization token. Here is how to obtain the token.
    • Data: This should contain a stringified, valid json object containing an array of objects with key "zoneId" and value of a valid secure zone id

    Sample request

    {% raw %}var access_token = BCAPI.Helper.Site.getAccessToken();
var request = $.ajax({
    url: "/webresources/api/v3/sites/current/news/87261/securezones",
    type: "POST",
    mimeType: "application/json", 
    connection: "keep-alive",
    contentType: "application/json",headers: {
        "Authorization": $.cookie('access_token')
    },
    processData: false,
    dataType: "html",
    data: JSON.stringify({
        "items": [{
            "zoneId": 51
        }]                    
    })
});

request.done(function (msg) {
    console.log(msg);
})

request.fail(function (jqXHR) {
    console.log("Request failed.");
    console.log("Error code: " + jqXHR.status);
    console.log("Error text: " + jqXHR.statusText);
    console.log("Response text: " + jqXHR.responseText);
}){% endraw %}

    Response

    Returns an empty body if successful or a detailed error message.

    Accepts and returns JSON as Content-Type.
