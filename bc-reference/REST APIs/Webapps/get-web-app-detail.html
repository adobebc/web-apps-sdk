Get Web App Detail

Get the details for a specific Web App.

Request

  • Method: GET
  • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the Authorize your API calls document for more info on how this URL is formed.
    • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
  • Path: /api/v2/admin/sites/current/webapps/[NAME]
    • Alternatively use siteID instead of 'current'
  • Authorization header: This should contain the authorization token. Here is how to obtain the token.
  • Required Permissions: Webapp View

Parameters

Use the name of the webapp as part of the URL (see Path)

Response

A webapp object with the following properties:

  • {% raw %}id{% endraw %} - the id of the webapp in the current site (integer)
  • {% raw %}name{% endraw %} - the name of the webapp (string)
  • {% raw %}slug{% endraw %} - the SEO friendly slug generated for the webapp (string)
  • {% raw %}allowFileUpload{% endraw %} - permits or denies file uploads to take place when saving a webapp item (boolean)
  • {% raw %}createBy{% endraw %} - the ID of the user that created the webapp. Details can be retrieved using the Admin Users API (integer)
  • {% raw %}createDate{% endraw %} - the date when the web app was created; uses the format yyyy-mm-dd (string)
  • {% raw %}lastUpdateBy{% endraw %} - the ID of the user that created the webapp. Details can be retrieved using the Admin Users API (integer)
  • {% raw %}lastUpdateDate{% endraw %} - the date when the web app item was created; uses the format yyyy-mm-dd (string)
  • {% raw %}disableDetailPages{% endraw %} - if True the Web App items detail page will not be accessible. The detail page will also be excluded from the site search feature built into this system as well as from the sitemap.xml file submitted to the major search engines such as Google, Yahoo and Live. (boolean)
  • {% raw %}locationEnabled{% endraw %} - use location for items (boolean)
  • {% raw %}anyoneCanEdit{% endraw %} - allow anyone to edit the items, not only the owner (boolean)
  • {% raw %}customerCanAdd{% endraw %} - users can add items in front-end (boolean)
  • {% raw %}customerCanEdit{% endraw %} - users can edit their own items (boolean)
  • {% raw %}customerCanDelete{% endraw %} - users can delete their own items (boolean)
  • {% raw %}requiresPayment{% endraw %} - users must pay to view this item (boolean)
  • {% raw %}requiresApproval{% endraw %} - items must be approved prior to becoming visible in the frontend (boolean)
  • {% raw %}roleId{% endraw %} - the id of the role assigned to the webapp (integer)
  • {% raw %}templateId{% endraw %} - the id of the template for the webapp (integer)
  • {% raw %}validDays{% endraw %} - how many days will the item remain visible. Default value is -1, which means never expire (integer)
  • {% raw %}uploadFolder{% endraw %} - path to the folder where files associated with items get uploaded to (string)
  • {% raw %}systemFields{% endraw %} - (array)
    • {% raw %}name{% endraw %} - name of the webapp item (string)
    • {% raw %}description{% endraw %} - the content in the webapp item (string)
    • {% raw %}weight{% endraw %} - the weight of the item; determines order of display (integer)
    • {% raw %}releaseDate{% endraw %} - the release date of the web app item; uses the format yyyy-mm-dd (string)
    • {% raw %}expiryDate{% endraw %} - the date when the web app item will expire; uses the format yyyy-mm-dd (string)
    • {% raw %}createDate{% endraw %} - the date when the web app item was created; uses the format yyyy-mm-dd (string)
    • {% raw %}lastUpdateDate{% endraw %} - the date when the webapp item was last updated; uses the format yyyy-mm-dd (string)
    • {% raw %}enabled{% endraw %} - indicates whether this web app item is enabled or disabled (boolean)
    • {% raw %}roleId{% endraw %} - id of the role assigned; inherits from the webapp (integer)
    • {% raw %}createBy{% endraw %} - id of the user that created the webapp item (integer)
    • {% raw %}submittedBy{% endraw %} - id of the user that submitted the webapp item (can be different, but the default is to equal the createBy) (integer)
    • {% raw %}address{% endraw %} - optional, only when enableLocation is True, stores the address associated with the item (string)
    • {% raw %}city{% endraw %} - optional, only when enableLocation is True, stores the city associated with the item (string)
    • {% raw %}state{% endraw %} - optional, only when enableLocation is True, stores the state associated with the item (string)
    • {% raw %}zipcode{% endraw %} - optional, only when enableLocation is True, stores the zipcode associated with the item; client-side validation for the zipcode format is recommended (string)
    • {% raw %}country{% endraw %} - optional, only when enableLocation is True, stores the country associated with the item (string)
  • {% raw %}fields{% endraw %} - a collection of all custom fields associated with the webapp. Added to the webapp object as shorthand for following the rel='fields' link mentioned below (array)
  • {% raw %}links{% endraw %} - a collection of uris are provided for discoverability. Each object contains links to: (array)
    • {% raw %}items{% endraw %}- points to the sub-api that lists all webapp items for a webapp
    • {% raw %}fields{% endraw %}- points to the api that allows management of the custom fields
    • {% raw %}previewUrl{% endraw %} - link to see the webapp in the site front-end. Composed from the site default domain and the slug
    • {% raw %}countries{% endraw %} - link to get the list of countries allowed for location, if location is enabled for the webapp

Error codes

This method will return the following error codes:

  • {% raw %}200{% endraw %} - success
  • {% raw %}401{% endraw %} - unauthorized - when the Authorization header is not present, or contains an invalid site token
    • {% raw %}101000{% endraw %} - sub-error code
  • {% raw %}403{% endraw %} - forbidden : this is returned when the user trying to access the API does not have the proper permissions
  • {% raw %}404{% endraw %} - if trying to access a webapp that does not exist
