Create Web App Custom Field

Create a new Web App custom field.

Request

  • Method: POST
  • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the Authorize your API calls document for more info on how this URL is formed.
    • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
  • Path: /api/v2/admin/sites/current/webapps/[webappName]/fields
  • Authorization header: This should contain the authorization token. Here is how to obtain the token.
  • Required Permissions: Edit Web Apps

Parameters

Object:

  • {% raw %}id{% endraw %} - unique, required - field id - (int)
  • {% raw %}name{% endraw %} - unique, required - the name of the field (it must be unique amongst the custom & system fields) (string)
  • {% raw %}type{% endraw %} - required, one of the below: (string)
    • {% raw %}String{% endraw %}
    • {% raw %}Number{% endraw %}
    • {% raw %}Boolean{% endraw %}
    • {% raw %}DateTime{% endraw %}
    • {% raw %}DropDown_List{% endraw %}
    • {% raw %}CheckBox_List{% endraw %}
    • {% raw %}Radio_List{% endraw %}
    • {% raw %}Image{% endraw %}
    • {% raw %}String_MultiLine{% endraw %}
    • {% raw %}ListBox_List{% endraw %}
    • {% raw %}String_Hyperlink{% endraw %}
    • {% raw %}DataSource{% endraw %}
  • {% raw %}required{% endraw %} - true if the field is required (not required) (boolean)

  • {% raw %}order{% endraw %} - unique - the position of the field when listed starting with 1.

    If a field with the same order value already exists, its order value (and order value for any other fields "bellow") will be incremented by 1.

    If not specified, the field is added as the last one. (integer)

  • {% raw %}dataSource{% endraw %}: for DataSource fields should be a valid webapp name (string)

  • {% raw %}listItems{% endraw %} - used only by DropDown_List, CheckBox_List, Radio_List, ListBox_List fields (array)
  • {% raw %}addEmptyOption{% endraw %} - choose to show a default "Please Select" option for {% raw %}DropDown_List{% endraw %} and {% raw %}DataSource{% endraw %} fields when editing Web App Items in Admin Console - applicable only for {% raw %}DropDown_List{% endraw %} and {% raw %}DataSource{% endraw %} fields - default is false (boolean)
  • {% raw %}useRichEditor{% endraw %} - define if the Rich Editor is to be used for {% raw %}String_MultiLine{% endraw %} fields when editing Web App items in Admin Console - default is false (boolean)

Response

The URI of the newly created custom field in the Location HTTP header.

Examples

Accepts and returns JSON as Content-Type.

Request:

{% raw %}POST /api/v2/admin/sites/current/webapps/WEBAPP_NAME_HERE/fields HTTPS/1.1
Authorization: 7e04176165cd4df7b628cc8ad022a8becf3bb84de44d480aadfae75e4fa07b56
Content-Length: 157
Connection: keep-alive
Content-Type: application/json

{
    "id": 44,
    "name": "Part code",
    "type": "DropDown_List",
    "listItems": [
        "First option",
        "Second one"
    ],
    "required": false,
    "order": 3
}
{% endraw %}

Response:

{% raw %}HTTP/1.1 201 OK
Content-Type: application/json; charset=utf-8
Location: /api/v2/admin/sites/current/webapps/Cars/fields/123
{% endraw %}

Sample code

var request = $.ajax({
    url: "/api/v2/admin/sites/current/webapps/test app2/fields",
    type: "POST",
    connection: "keep-alive",
    contentType: "application/json",
    headers: {
        "Authorization": $.cookie('access_token')
    },
    processData: false,
    data: JSON.stringify({
        "name": "Customfield1",
        "type": "CheckBox_List",
        "required": true,
        "order": 1,
        "listItems": ["First option", "Second one"],
        "addEmptyOption": true
    })
});
request.done(function(msg) {
    console.log(msg);
});
request.fail(function(jqXHR) {
    console.log("Request failed.");
    console.log("Error code: " + jqXHR.status);
    console.log("Error text: " + jqXHR.statusText);
    console.log("Response text: " + jqXHR.responseText);
});

Error codes

This method will return the following error codes:

  • {% raw %}201{% endraw %} - created
  • {% raw %}400{% endraw %} - bad request; sub-error codes:
    • {% raw %}210000{% endraw %} - An unspecified error has occurred
    • {% raw %}210002{% endraw %} - Invalid field id specified
    • {% raw %}210003{% endraw %} - Invalid name (name is not syntactically correct or there is already a system or custom field with the same name)
    • {% raw %}210004{% endraw %} - Invalid listItems
    • {% raw %}210005{% endraw %} - Invalid dataSourceName
    • {% raw %}210006{% endraw %} - Invalid order value (must be between 1 and max existing order value + 1)
    • {% raw %}210007{% endraw %} - Invalid type
  • {% raw %}401{% endraw %} - unauthorized - when the authentication token is incorrect
    • {% raw %}101000{% endraw %} - sub-error code
  • {% raw %}403{% endraw %} - forbidden - when the user does not have Edit WebApp Permission
  • {% raw %}404{% endraw %} - not found
    • {% raw %}190001{% endraw %} - The web app could not be found (the webAppName param from the URL does not match)
    • {% raw %}210001{% endraw %} - The custom field was not found (the fieldId from the URL do not match)
