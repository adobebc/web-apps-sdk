{module_contentholder}

Description

Displays the individual content holder. This module also supports named content holders.

This module also supports Liquid, take a look at this article to learn how to access its Liquid output tags and render it using your own custom logic.

  • New named parameters documentation
  • Legacy module documentation

Syntax

{% raw %}{module_contentholder,name="contentholder_name" collection="my_custom_collection_name" template=""}{% endraw %}

Note: When using named parameters always wrap the individual parameters in double quotes.

Parameters

  • ID - system generated (do not change)

or

  • name - is this represents the contentholder's name
  • collection - use this if you need to access the module's Liquid output tags
  • template - this module does not support custom templates. You can use the template parameter however to supress the module's output

This module can be rendered:

  • in plain text
  • Liquid collections - here are the Liquid output tags you can use:
Liquid tag Description
{% raw %}{{id}}{% endraw %} The content holder's ID
{% raw %}{{name}}{% endraw %} The content holder's name
{% raw %}{{content}}{% endraw %} The content holder's content

Examples

  • {% raw %}{module_contentholder,34234}{% endraw %} - will display the content holder with the ID 34234
  • {% raw %}{module_contentholder,name="Example"}{% endraw %} - renders the Content Holder with name "Example" Where the SFTP file is Example.html
  • {% raw %}{module_contentholder,name="{system_visitordeviceclass}"{% endraw %} - depending on the type of the device the site visitor is using it renders the content holder with the names desktop, phone or tablet. See the Deploying your site across Mobile and Tablet devices document for more details on this setup
  • {% raw %}module_contentholder name="Addresses" collection="collection1" template=""}{% endraw %} - in this case nothing will be displayed, however the module's Liquid tags will become available in the collection1 collection.
